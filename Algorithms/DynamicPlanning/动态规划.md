## 一、什么是动态规划

### 1.1 一句话定义

> **动态规划 = 暴力搜索 + 记忆化（缓存）+ 重叠子问题消除**

DP 的本质是：把一个大问题拆成若干**相互关联的子问题**，先求解子问题，
再通过子问题的答案组合出大问题的答案，避免重复计算。

### 1.2 DP vs 其他算法思想

| 特征 | 暴力递归 | 记忆化搜索（自顶向下） | DP 迭代（自底向上） | 贪心 |
|------|---------|----------------------|-------------------|------|
| 是否有重叠子问题 | 有，重复计算 | 有，用缓存消除 | 有，按顺序计算 | 无 |
| 是否有最优子结构 | 是 | 是 | 是 | 是（更严格） |
| 决策方式 | 穷举所有 | 穷举 + 缓存 | 穷举 + 填表 | 每步选局部最优 |
| 时间复杂度 | 指数级 | 通常多项式 | 通常多项式 | 通常最优 |

**关键区分**：能用贪心的问题一定可以用 DP，但能用 DP 的问题不一定能用贪心。
贪心需要更强的条件（局部最优 → 全局最优），DP 则是暴力枚举所有可能再取最优。

### 1.3 什么时候该想到 DP？

当你看到以下关键词时，优先考虑 DP：

- **"最大/最小"**：最长、最短、最多、最少、最大利润……
- **"是否可能"**：能否恰好凑出某个值……
- **"方案数"**：有多少种方式达到目标……
- **"序列/字符串"**：子序列、子串匹配……
- **"网格/路径"**：从左上到右下的路径数……

## 二、DP 的两大核心要素

### 2.1 最优子结构（Optimal Substructure）

> 大问题的最优解包含子问题的最优解。

**示例**：爬楼梯——到达第 `n` 阶的方案数 = 到达第 `n-1` 阶 + 到达第 `n-2` 阶的方案数。

### 2.2 无后效性（No Aftereffect）

> 一旦某个状态被确定，它之前的历史过程不影响后续决策。

**示例**：买卖股票——当前持有的最低成本只取决于历史最低价，不关心是哪天买的。

### 2.3 重叠子问题（Overlapping Subproblems）

> 递归过程中，同一个子问题被多次求解。

**经典例子**：斐波那契数列 `fib(5)` 的递归树：

```
                 fib(5)
                /      \
           fib(4)      fib(3)       ← fib(3) 被计算了 2 次
           /    \       /    \
       fib(3) fib(2) fib(2) fib(1)  ← fib(2) 被计算了 3 次
       /   \
   fib(2) fib(1)
```

DP 通过缓存或按序填表，让每个子问题只计算一次。

## 三、DP 问题的通用解题框架

遇到一道新题，按以下 **5 步法** 思考：

### 第 1 步：定义状态（State Definition）⭐⭐⭐

> **这是最关键的一步，也是最难的一步。**

问自己：**"为了描述整个问题的求解过程，我需要记录哪些信息？"**

状态就是子问题的"签名"。好的状态定义应该满足：
- **完整性**：能通过状态唯一确定一个子问题
- **无后效性**：状态一旦确定，后续决策与到达该状态的路径无关

**常见状态定义模式**：

| 问题类型 | 状态 `dp[i]` 的含义 |
|---------|-------------------|
| 一维序列 | `dp[i]` = 以 `i` 结尾（或前 `i` 个元素）的最优解 |
| 二维网格 | `dp[i][j]` = 到达 `(i, j)` 的最优解 |
| 背包问题 | `dp[i][w]` = 前 `i` 件物品、容量为 `w` 时的最优解 |
| 字符串匹配 | `dp[i][j]` = `s1[0..i]` 与 `s2[0..j]` 的匹配结果 |
| 区间问题 | `dp[i][j]` = 区间 `[i, j]` 的最优解 |

### 第 2 步：写出状态转移方程（Transition）⭐⭐⭐

> **这是 DP 的灵魂。**

问自己：**"`dp[i]` 能从哪些更小的状态转移过来？"**

**示例**（爬楼梯）：
```
dp[i] = dp[i-1] + dp[i-2]
       ↑ 从第 i-1 阶迈 1 步    ↑ 从第 i-2 阶迈 2 步
```

**技巧**：
- 从最后一步倒推："最后一步做了什么？做了这一步之前的状态是什么？"
- 列举所有可能的"最后一步"，取最优

### 第 3 步：确定初始条件（Base Case）

> 状态转移方程无法再递推时的终止状态。

**示例**：
```
dp[0] = 1  （站在第 0 阶，1 种方案：不动）
dp[1] = 1  （站在第 1 阶，1 种方案：从 0 迈 1 步）
```

**常见错误**：初始条件写错 → 整个 DP 表全错。务必手动验证小规模用例。

### 第 4 步：确定遍历顺序（Traversal Order）

> 保证计算 `dp[i]` 时，它依赖的状态已经计算完毕。

| 转移方向 | 遍历顺序 |
|---------|---------|
| `dp[i]` 依赖 `dp[i-1]` | `i` 从小到大 |
| `dp[i]` 依赖 `dp[i+1]` | `i` 从大到小 |
| `dp[i][j]` 依赖 `dp[i-1][j]` 和 `dp[i][j-1]` | 先行后列（或先列后行） |
| `dp[i][j]` 依赖 `dp[i+1][j-1]` | 需要按区间长度枚举 |

### 第 5 步：返回结果（Result）

> 最终答案不一定是 `dp[n]`。

常见情况：
- `dp[n]` 或 `dp[n-1]`：整个序列的最优解
- `max(dp[i])` 或 `min(dp[i])`：所有位置中的最优解
- `dp[0][n-1]`：整个区间的最优解

## 四、DP 的常见分类与经典问题

### 4.1 一维线性 DP

状态只与一个维度（通常是下标 `i`）有关。

#### 经典问题 1：爬楼梯（LeetCode 70）

> 一次可以上 1 级或 2 级台阶，到达第 `n` 级有多少种方案？

```python
def climbStairs(n: int) -> int:
    """
    状态定义：dp[i] = 到达第 i 级台阶的方案数
    转移方程：dp[i] = dp[i-1] + dp[i-2]
    初始条件：dp[0] = 1, dp[1] = 1
    遍历顺序：i 从小到大
    """
    if n <= 1:
        return 1
    dp = [0] * (n + 1)
    dp[0], dp[1] = 1, 1
    for i in range(2, n + 1):
        dp[i] = dp[i - 1] + dp[i - 2]
    return dp[n]
```

#### 经典问题 2：打家劫舍（LeetCode 198）

> 不能同时抢相邻的两间房子，求最大收益。

```python
def rob(nums: list[int]) -> int:
    """
    状态定义：dp[i] = 抢前 i 间房子的最大收益
    转移方程：dp[i] = max(dp[i-1], dp[i-2] + nums[i])
                        ↑ 不抢第 i 间     ↑ 抢第 i 间
    初始条件：dp[0] = nums[0], dp[1] = max(nums[0], nums[1])
    """
    if len(nums) == 1:
        return nums[0]
    n = len(nums)
    dp = [0] * n
    dp[0] = nums[0]
    dp[1] = max(nums[0], nums[1])
    for i in range(2, n):
        dp[i] = max(dp[i - 1], dp[i - 2] + nums[i])
    return dp[-1]
```

### 4.2 二维网格 DP

状态由两个维度 `(i, j)` 描述，通常用于网格类问题。

#### 经典问题：不同路径（LeetCode 62）

> 从 `(0,0)` 走到 `(m-1,n-1)`，每次只能向右或向下，有多少种走法？

```python
def uniquePaths(m: int, n: int) -> int:
    """
    状态定义：dp[i][j] = 从 (0,0) 到达 (i,j) 的路径数
    转移方程：dp[i][j] = dp[i-1][j] + dp[i][j-1]
    初始条件：dp[0][j] = 1, dp[i][0] = 1（边界只有一条路径）
    遍历顺序：先行后列（从左到右，从上到下）
    """
    dp = [[1] * n for _ in range(m)]
    for i in range(1, m):
        for j in range(1, n):
            dp[i][j] = dp[i - 1][j] + dp[i][j - 1]
    return dp[m - 1][n - 1]
```

**空间优化**：由于 `dp[i][j]` 只依赖 `dp[i-1][j]` 和 `dp[i][j-1]`，
可以用一维滚动数组将空间从 `O(m*n)` 优化到 `O(n)`：

```python
def uniquePaths_optimized(m: int, n: int) -> int:
    dp = [1] * n
    for i in range(1, m):
        for j in range(1, n):
            dp[j] += dp[j - 1]  # dp[j] 保留了上一行的值，dp[j-1] 是当前行已更新的值
    return dp[n - 1]
```

### 4.3 背包问题

背包问题是 DP 中最经典的模型，理解它能解决大量变体问题。

#### 4.3.1 0-1 背包

> 有 `n` 件物品和一个容量为 `W` 的背包。第 `i` 件物品重量 `w[i]`、价值 `v[i]`。
> 每件物品只能选一次，求最大总价值。

```python
def knapsack_01(W: int, weights: list[int], values: list[int]) -> int:
    """
    状态定义：dp[i][w] = 前 i 件物品、背包容量为 w 时的最大价值
    转移方程：
        dp[i][w] = max(
            dp[i-1][w],              ← 不选第 i 件
            dp[i-1][w-w[i]] + v[i]   ← 选第 i 件（前提：w >= w[i]）
        )
    初始条件：dp[0][w] = 0（没有物品可选时价值为 0）
    遍历顺序：物品 i 从小到大，容量 w 从小到大
    """
    n = len(weights)
    dp = [[0] * (W + 1) for _ in range(n + 1)]

    for i in range(1, n + 1):
        for w in range(W + 1):
            dp[i][w] = dp[i - 1][w]  # 不选第 i 件
            if w >= weights[i - 1]:
                dp[i][w] = max(dp[i][w], dp[i - 1][w - weights[i - 1]] + values[i - 1])

    return dp[n][W]
```

**一维空间优化**（逆序遍历容量！）：

```python
def knapsack_01_optimized(W: int, weights: list[int], values: list[int]) -> int:
    """
    关键：容量 w 必须**从大到小**遍历，保证每件物品只被选一次。
    如果从小到大遍历，就变成了完全背包（每件物品可重复选）。
    """
    dp = [0] * (W + 1)
    for i in range(len(weights)):
        for w in range(W, weights[i] - 1, -1):  # 逆序！
            dp[w] = max(dp[w], dp[w - weights[i]] + values[i])
    return dp[W]
```

#### 4.3.2 完全背包

> 与 0-1 背包的区别：每件物品可以**无限次**选取。

```python
def knapsack_complete(W: int, weights: list[int], values: list[int]) -> int:
    """
    唯一区别：容量 w 从小到大遍历（正序），允许重复选取。
    """
    dp = [0] * (W + 1)
    for i in range(len(weights)):
        for w in range(weights[i], W + 1):  # 正序！
            dp[w] = max(dp[w], dp[w - weights[i]] + values[i])
    return dp[W]
```

#### 4.3.3 背包问题速查表

| 类型 | 每件物品选取次数 | 内层循环方向 | 状态转移 |
|------|----------------|-------------|---------|
| 0-1 背包 | 最多 1 次 | 容量**从大到小** | `dp[w] = max(dp[w], dp[w-wi]+vi)` |
| 完全背包 | 无限次 | 容量**从小到大** | `dp[w] = max(dp[w], dp[w-wi]+vi)` |
| 多重背包 | 最多 `k` 次 | 拆分为 0-1 背包或二进制优化 | 类似 0-1 背包 |

### 4.4 子序列问题

子序列问题是面试高频考点，需要重点掌握。

#### 4.4.1 最长递增子序列 LIS（LeetCode 300）

> 给定数组，找到最长严格递增子序列的长度。

**解法一：DP（O(n²)）**

```python
def lengthOfLIS(nums: list[int]) -> int:
    """
    状态定义：dp[i] = 以 nums[i] 结尾的最长递增子序列长度
    转移方程：dp[i] = max(dp[j] + 1) for all j < i where nums[j] < nums[i]
    含义：在所有比 nums[i] 小的元素中，找到 dp[j] 最大的那个，接在后面
    """
    n = len(nums)
    dp = [1] * n  # 每个元素自身构成长度为 1 的子序列

    for i in range(1, n):
        for j in range(i):
            if nums[j] < nums[i]:
                dp[i] = max(dp[i], dp[j] + 1)

    return max(dp)
```

**解法二：贪心 + 二分（O(n log n)）**

```python
import bisect

def lengthOfLIS_optimized(nums: list[int]) -> int:
    """
    维护一个数组 tails，tails[k] = 长度为 k+1 的递增子序列的最小末尾元素。
    对于每个新元素 num：
    - 如果 num > tails[-1]，直接追加（延长子序列）
    - 否则，在 tails 中二分查找第一个 >= num 的位置并替换（保持 tails 尽量小）
    """
    tails = []
    for num in nums:
        pos = bisect.bisect_left(tails, num)
        if pos == len(tails):
            tails.append(num)
        else:
            tails[pos] = num
    return len(tails)
```

#### 4.4.2 最长公共子序列 LCS（LeetCode 1143）

> 给定两个字符串，求它们的最长公共子序列长度。

```python
def longestCommonSubsequence(text1: str, text2: str) -> int:
    """
    状态定义：dp[i][j] = text1[0..i-1] 与 text2[0..j-1] 的 LCS 长度
    转移方程：
        if text1[i-1] == text2[j-1]:
            dp[i][j] = dp[i-1][j-1] + 1     ← 字符匹配，LCS 长度 +1
        else:
            dp[i][j] = max(dp[i-1][j], dp[i][j-1])  ← 不匹配，跳过其中一个
    """
    m, n = len(text1), len(text2)
    dp = [[0] * (n + 1) for _ in range(m + 1)]

    for i in range(1, m + 1):
        for j in range(1, n + 1):
            if text1[i - 1] == text2[j - 1]:
                dp[i][j] = dp[i - 1][j - 1] + 1
            else:
                dp[i][j] = max(dp[i - 1][j], dp[i][j - 1])

    return dp[m][n]
```

#### 4.4.3 子序列问题状态定义速查

| 问题 | 状态 `dp[i]` 或 `dp[i][j]` 的含义 |
|------|--------------------------------|
| LIS | `dp[i]` = 以 `nums[i]` 结尾的 LIS 长度 |
| LCS | `dp[i][j]` = `s1[0..i-1]` 与 `s2[0..j-1]` 的 LCS 长度 |
| 编辑距离 | `dp[i][j]` = `s1[0..i-1]` 变换为 `s2[0..j-1]` 的最少操作数 |
| 回文子串 | `dp[i][j]` = `s[i..j]` 是否为回文 |

### 4.5 区间 DP

> 在一个区间 `[i, j]` 上进行决策，将区间拆分为两个子区间。

#### 经典问题：戳气球（LeetCode 312）

> 戳破所有气球，每次戳破一个获得 `nums[left] * nums[i] * nums[right]` 的硬币。

```python
def maxCoins(nums: list[int]) -> int:
    """
    状态定义：dp[i][j] = 戳破开区间 (i, j) 内所有气球能获得的最大硬币数
    转移方程：枚举 (i, j) 内最后一个被戳破的气球 k：
        dp[i][j] = max(dp[i][k] + dp[k][j] + nums[i]*nums[k]*nums[j])
    关键：逆向思考——不是"先戳哪个"，而是"最后戳哪个"
    """
    n = len(nums)
    # 首尾添加虚拟气球 1，简化边界处理
    balloons = [1] + nums + [1]
    dp = [[0] * (n + 2) for _ in range(n + 2)]

    # 按区间长度从小到大枚举
    for length in range(2, n + 2):          # 区间长度（至少 2 个端点）
        for i in range(0, n + 2 - length):  # 左端点
            j = i + length                   # 右端点
            for k in range(i + 1, j):        # 最后戳破的气球 k
                dp[i][j] = max(
                    dp[i][j],
                    dp[i][k] + dp[k][j] + balloons[i] * balloons[k] * balloons[j]
                )

    return dp[0][n + 1]
```

**区间 DP 遍历模板**：
```python
# 按区间长度枚举
for length in range(2, n + 1):
    for i in range(n - length + 1):
        j = i + length - 1
        for k in range(i, j):  # 枚举分割点
            dp[i][j] = max/min(dp[i][j], dp[i][k] + dp[k+1][j] + cost)
```

### 4.6 状态机 DP

> 每个状态代表一种"模式"，在不同模式之间转换。

#### 经典问题：买卖股票的最佳时机 II（LeetCode 122）

> 可以进行多次交易，求最大利润。

```python
def maxProfit(prices: list[int]) -> int:
    """
    状态定义：
        hold = 当前持有股票时的最大利润
        cash = 当前不持有股票时的最大利润

    转移方程：
        hold = max(hold, cash - prices[i])   ← 继续持有 or 今天买入
        cash = max(cash, hold + prices[i])   ← 继续不持有 or 今天卖出
    """
    hold = float('-inf')
    cash = 0

    for price in prices:
        hold = max(hold, cash - price)
        cash = max(cash, hold + price)

    return cash
```

**状态机 DP 模板**：
```
            ┌─ 买入 ─┐
            ↓        │
         [持有] ──卖出──→ [未持有]
            │              ↑
            └── 继续持有 ──┘
```

### 4.7 树形 DP

> 在树结构上进行动态规划，通常需要在 DFS 过程中自底向上传递信息。

#### 经典问题：二叉树的最大路径和（LeetCode 124）

```python
class TreeNode:
    def __init__(self, val=0, left=None, right=None):
        self.val = val
        self.left = left
        self.right = right

def maxPathSum(root: TreeNode) -> int:
    """
    对于每个节点，计算"经过该节点的最大路径和"：
        经过 node 的最大路径和 = node.val + max(左子树贡献, 0) + max(右子树贡献, 0)

    返回给父节点的值（单边路径）：
        node.val + max(左子树贡献, 右子树贡献, 0)
    """
    result = float('-inf')

    def dfs(node: TreeNode) -> int:
        """返回以 node 为端点的最大单边路径和"""
        nonlocal result
        if not node:
            return 0

        left = max(dfs(node.left), 0)    # 左子树贡献（负数不如不要）
        right = max(dfs(node.right), 0)  # 右子树贡献

        # 更新全局最大路径和（经过当前节点的路径）
        result = max(result, node.val + left + right)

        # 返回给父节点：只能选一边
        return node.val + max(left, right)

    dfs(root)
    return result
```

### 4.8 数位 DP

> 统计满足某种性质的数字个数，通常按位枚举。

#### 经典问题：统计各位数字之和等于 target 的数字个数（范围 [1, n]）

```python
def countDigitSum(n: int, target: int) -> int:
    """
    数位 DP 三要素：
        1. 数字的位数（从高位到低位枚举）
        2. 当前数位之和
        3. 是否受到上界限制（tight）

    状态定义：dfs(pos, sum_so_far, tight)
        pos        = 当前处理的位数（从高到低）
        sum_so_far = 已选数位之和
        tight      = 是否受到上界限制
    """
    digits = [int(d) for d in str(n)]
    length = len(digits)

    from functools import lru_cache

    @lru_cache(maxsize=None)
    def dfs(pos: int, digit_sum: int, tight: bool) -> int:
        if digit_sum > target:
            return 0
        if pos == length:
            return 1 if digit_sum == target else 0

        limit = digits[pos] if tight else 9
        result = 0
        for d in range(0, limit + 1):
            result += dfs(pos + 1, digit_sum + d, tight and d == limit)

        return result

    return dfs(0, 0, True)
```

## 五、DP 优化技巧

### 5.1 滚动数组（空间优化）

当 `dp[i]` 只依赖 `dp[i-1]` 时，可以用两个变量交替更新：

```python
# 优化前：O(n) 空间
dp = [0] * (n + 1)
dp[0], dp[1] = 1, 1
for i in range(2, n + 1):
    dp[i] = dp[i-1] + dp[i-2]

# 优化后：O(1) 空间
prev, curr = 1, 1
for i in range(2, n + 1):
    prev, curr = curr, prev + curr
```

### 5.2 状态压缩（位运算）

当状态集合可以用二进制表示时，用整数代替集合：

```python
# 状态 dp[mask] 表示已经访问过的节点集合为 mask
# 例：mask = 0b1010 表示访问了第 1 和第 3 个节点

# 添加节点 i 到集合
new_mask = mask | (1 << i)

# 检查节点 i 是否在集合中
if mask & (1 << i):
    ...
```

### 5.3 单调队列 / 单调栈优化

当状态转移涉及"滑动窗口最值"时，可以用单调队列将 O(nk) 优化到 O(n)：

```python
from collections import deque

def maxSlidingWindow(nums: list[int], k: int) -> list[int]:
    """单调队列优化示例"""
    dq = deque()  # 存放下标，保持队列中元素从大到小
    result = []
    for i in range(len(nums)):
        # 移除超出窗口的元素
        while dq and dq[0] < i - k + 1:
            dq.popleft()
        # 保持单调递减
        while dq and nums[dq[-1]] <= nums[i]:
            dq.pop()
        dq.append(i)
        if i >= k - 1:
            result.append(nums[dq[0]])
    return result
```

### 5.4 前缀和优化

当转移方程涉及"区间和"时，预计算前缀和：

```python
def subarraySum(nums: list[int], k: int) -> int:
    """利用前缀和 + 哈希表，O(n) 统计和为 k 的子数组个数"""
    count = 0
    prefix_sum = 0
    seen = {0: 1}  # 前缀和 -> 出现次数

    for num in nums:
        prefix_sum += num
        if prefix_sum - k in seen:
            count += seen[prefix_sum - k]
        seen[prefix_sum] = seen.get(prefix_sum, 0) + 1

    return count
```

## 七、常见误区与避坑指南

### ❌ 误区 1：直接背状态转移方程

**正确做法**：理解"最后一步"的决策逻辑，方程自然就写出来了。

### ❌ 误区 2：忽略初始条件

**正确做法**：写完转移方程后，立即写出 base case，并用小规模用例手动验证。

### ❌ 误区 3：遍历顺序错误

**正确做法**：画出依赖关系图，确认计算 `dp[i]` 时它依赖的状态已经算出。

### ❌ 误区 4：忘记空间优化

**正确做法**：当 `dp[i]` 只依赖 `dp[i-1]` 时，考虑用变量代替数组。

### ❌ 误区 5：混淆 0-1 背包和完全背包

**正确做法**：
- 0-1 背包：内层循环**从大到小**（逆序）
- 完全背包：内层循环**从小到大**（正序）

### ❌ 误区 6：强行用 DP

**正确做法**：如果问题满足贪心选择性质，优先用贪心（更高效）。
只有当贪心无法保证全局最优时，才用 DP。
