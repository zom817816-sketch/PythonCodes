# 字符串
## 字符串匹配算法
[力扣题目链接](https://leetcode.cn/problems/find-the-index-of-the-first-occurrence-in-a-string/)
```
实现 strStr() 函数。
给定一个 haystack 字符串和一个 needle 字符串，在 haystack 字符串中找出 needle 字符串出现的第一个位置 (从0开始)。如果不存在，则返回  -1。
示例 1: 输入: haystack = "hello", needle = "ll" 输出: 2
示例 2: 输入: haystack = "aaaaa", needle = "bba" 输出: -1
说明: 当 needle 是空字符串时，我们应当返回什么值呢？这是一个在面试中很好的问题。 对于本题而言，当 needle 是空字符串时我们应当返回 0 。这与C语言的 strstr() 以及 Java的 indexOf() 定义相符。
```
### 暴力匹配算法（Brute Force）
#### 核心思想
逐个比较文本串和模式串的每个字符，当发现不匹配时，将模式串向右移动一位重新开始比较。

#### 时间复杂度
- 最坏情况：O(n×m)，其中n是文本串长度，m是模式串长度
- 最好情况：O(n)，当模式串在文本串开头时
- 平均情况：O(n×m)

#### Python实现
```python
def strStr_brute_force(haystack: str, needle: str) -> int:
    if not needle:
        return 0
    
    n, m = len(haystack), len(needle)
    
    # 如果模式串比文本串长，直接返回-1
    if m > n:
        return -1
    
    # 只需要遍历到 n-m 的位置
    for i in range(n - m + 1):
        # 检查当前位置是否匹配
        match = True
        for j in range(m):
            if haystack[i + j] != needle[j]:
                match = False
                break
        if match:
            return i
    
    return -1
```

#### 简洁实现（使用切片）
```python
def strStr_simple(haystack: str, needle: str) -> int:
    if not needle:
        return 0
    
    n, m = len(haystack), len(needle)
    
    for i in range(n - m + 1):
        if haystack[i:i+m] == needle:
            return i
    
    return -1
```

### KMP算法（Knuth-Morris-Pratt）
#### 核心思想 
当匹配失败时，不是简单地将模式串向后移动一位重新开始，而是利用已经匹配的部分信息，跳过一些肯定不会匹配的位置。
KMP算法观察到，当在模式串的第j个字符不匹配时，前面已经匹配的j-1个字符是已知的。我们可以利用这个信息来决定模式串应该向右移动多少位，而不是只移动一位。

#### 前缀表的构建
- **前缀**：不包含最后一个字符的所有连续子串
- **后缀**：不包含第一个字符的所有连续子串
- **部分匹配值**：对于位置i，子串`pattern[0:i+1]`中最长的相同前缀和后缀的长度

#### 前缀表示例
模式串 "ABCDABD" 的部分匹配表：
- 位置0: "A" → 0
- 位置1: "AB" → 0  
- 位置2: "ABC" → 0
- 位置3: "ABCD" → 0
- 位置4: "ABCDA" → 1（前缀"A" = 后缀"A"）
- 位置5: "ABCDAB" → 2（前缀"AB" = 后缀"AB"）
- 位置6: "ABCDABD" → 0

部分匹配表：`[0, 0, 0, 0, 1, 2, 0]`

#### 构建前缀表的算法
```python
def build_prefix_table(pattern):
    n = len(pattern)
    prefix = [0] * n
    length = 0
    i = 1
    
    while i < n:
        if pattern[i] == pattern[length]:
            length += 1
            prefix[i] = length
            i += 1
        else:
            if length != 0:
                length = prefix[length - 1]
            else:
                prefix[i] = 0
                i += 1
    return prefix
```

#### KMP算法实现
```python
def strStr_KMP(haystack: str, needle: str) -> int:
    if not needle:
        return 0
    
    # 构建部分匹配表
    def build_prefix_table(pattern):
        n = len(pattern)
        prefix = [0] * n
        length = 0
        i = 1
        
        while i < n:
            if pattern[i] == pattern[length]:
                length += 1
                prefix[i] = length
                i += 1
            else:
                if length != 0:
                    length = prefix[length - 1]
                else:
                    prefix[i] = 0
                    i += 1
        return prefix
    
    prefix_table = build_prefix_table(needle)
    i = j = 0
    n, m = len(haystack), len(needle)
    
    while i < n:
        if haystack[i] == needle[j]:
            i += 1
            j += 1
            
            if j == m:
                return i - j
        else:
            if j != 0:
                j = prefix_table[j - 1]
            else:
                i += 1
    
    return -1
```

#### 时间复杂度
- 构建部分匹配表：O(m)
- 匹配过程：O(n)
- 总时间复杂度：O(n + m)

#### 空间复杂度
- O(m)，用于存储部分匹配表

### Boyer-Moore算法
#### 核心思想
从模式串的末尾开始比较，利用坏字符规则（Bad Character Rule）和好后缀规则（Good Suffix Rule）跳过大量不必要的比较。

#### 两个关键规则
1. **坏字符规则**：当发现不匹配的字符（坏字符）时，根据该字符在模式串中最后出现的位置来决定移动距离。
2. **好后缀规则**：当发现部分匹配的后缀时，根据该后缀在模式串中其他位置的出现情况来决定移动距离。

#### 算法步骤
1. 预处理模式串，构建坏字符表和好后缀表
2. 从模式串末尾开始比较
3. 当发现不匹配时，根据两个规则计算移动距离，取较大值
4. 重复直到找到匹配或遍历完文本串

#### Boyer-Moore算法实现
```python
def strStr_BoyerMoore(haystack: str, needle: str) -> int:
    if not needle:
        return 0
    
    n, m = len(haystack), len(needle)
    if m > n:
        return -1
    
    # 坏字符表：记录每个字符在模式串中最后出现的位置
    bad_char = {}
    for i in range(m):
        bad_char[needle[i]] = i
    
    # 好后缀表（简化版）
    good_suffix = [0] * (m + 1)
    suffix = [0] * (m + 1)
    
    # 预处理好后缀
    i = m
    j = m + 1
    suffix[i] = j
    
    while i > 0:
        while j <= m and needle[i - 1] != needle[j - 1]:
            if good_suffix[j] == 0:
                good_suffix[j] = j - i
            j = suffix[j]
        i -= 1
        j -= 1
        suffix[i] = j
    
    j = suffix[0]
    for i in range(m + 1):
        if good_suffix[i] == 0:
            good_suffix[i] = j
        if i == j:
            j = suffix[j]
    
    # 搜索过程
    s = 0
    while s <= n - m:
        j = m - 1
        
        # 从后向前比较
        while j >= 0 and needle[j] == haystack[s + j]:
            j -= 1
        
        if j < 0:
            # 找到匹配
            return s
        else:
            # 坏字符规则
            bc_shift = j - bad_char.get(haystack[s + j], -1)
            # 好后缀规则
            gs_shift = good_suffix[j + 1]
            
            # 取较大值作为移动距离
            s += max(1, bc_shift, gs_shift)
    
    return -1
```

#### 时间复杂度
- 最好情况：O(n/m)，当模式串不在文本串中且字符集较大时
- 最坏情况：O(n×m)，但实际应用中很少出现
- 平均情况：在实际文本搜索中非常高效

#### 空间复杂度
- O(m + σ)，其中σ是字符集大小

#### 适用场景
- 文本编辑器中的查找功能
- 大文件搜索
- 模式串较长的情况

### Rabin-Karp算法
#### 核心思想
使用哈希函数来快速比较字符串，通过滚动哈希（Rolling Hash）实现高效匹配。当哈希值匹配时，再进行实际的字符串比较以避免哈希冲突。

#### 关键概念
1. **哈希函数**：将字符串映射为一个数值
2. **滚动哈希**：当窗口滑动时，可以快速计算新窗口的哈希值，而不需要重新计算整个字符串
3. **模运算**：使用大质数作为模数来避免哈希值溢出

#### 算法步骤
1. 计算模式串的哈希值
2. 计算文本串第一个窗口的哈希值
3. 滑动窗口，使用滚动哈希更新哈希值
4. 当哈希值匹配时，进行实际的字符串比较
5. 重复直到找到匹配或遍历完文本串

#### Rabin-Karp算法实现
```python
def strStr_RabinKarp(haystack: str, needle: str) -> int:
    if not needle:
        return 0
    
    n, m = len(haystack), len(needle)
    if m > n:
        return -1
    
    # 选择一个质数作为模数
    MOD = 10**9 + 7
    # 基数（字符集大小）
    BASE = 256
    
    # 计算needle的哈希值和BASE^m % MOD
    needle_hash = 0
    haystack_hash = 0
    power = 1
    
    for i in range(m):
        needle_hash = (needle_hash * BASE + ord(needle[i])) % MOD
        haystack_hash = (haystack_hash * BASE + ord(haystack[i])) % MOD
        if i > 0:
            power = (power * BASE) % MOD
    
    # 检查第一个窗口
    if needle_hash == haystack_hash and haystack[:m] == needle:
        return 0
    
    # 滑动窗口
    for i in range(m, n):
        # 移除最左边的字符
        haystack_hash = (haystack_hash - ord(haystack[i - m]) * power) % MOD
        # 确保哈希值为正
        haystack_hash = (haystack_hash + MOD) % MOD
        # 添加新字符
        haystack_hash = (haystack_hash * BASE + ord(haystack[i])) % MOD
        
        # 检查哈希值匹配和实际字符串匹配
        if needle_hash == haystack_hash and haystack[i - m + 1:i + 1] == needle:
            return i - m + 1
    
    return -1
```

#### 时间复杂度
- 平均情况：O(n + m)
- 最坏情况：O(n×m)，当哈希冲突频繁时
- 最好情况：O(n + m)

#### 空间复杂度
- O(1)，只需要常数空间

#### 优点
1. **易于扩展**：可以同时搜索多个模式串
2. **适合流式数据**：可以处理数据流中的模式匹配
3. **实现相对简单**

#### 缺点
1. **哈希冲突**：可能需要额外的字符串比较
2. **需要选择好的哈希函数**：避免频繁的哈希冲突

#### 适用场景
- 多模式匹配
- 抄袭检测
- 流式数据中的模式匹配
- DNA序列匹配

## 算法比较

| 算法 | 时间复杂度 | 空间复杂度 | 优点 | 缺点 | 适用场景 |
|------|------------|------------|------|------|----------|
| **暴力匹配** | O(n×m) | O(1) | 实现简单，无需预处理 | 效率低，最坏情况差 | 短字符串、简单场景 |
| **KMP** | O(n+m) | O(m) | 最坏情况也是线性时间，文本指针不回溯 | 需要额外空间，实现复杂 | 文本编辑器、DNA序列、有重复模式的字符串 |
| **Boyer-Moore** | 最好O(n/m)，最坏O(n×m) | O(m+σ) | 实际应用中非常高效，特别适合长模式串 | 实现复杂，需要构建两个表 | 文本搜索、大文件搜索、模式串较长 |
| **Rabin-Karp** | 平均O(n+m)，最坏O(n×m) | O(1) | 易于扩展多模式匹配，适合流式数据 | 可能有哈希冲突，需要选择好的哈希函数 | 多模式匹配、抄袭检测、流式数据 |

## 实际应用建议

1. **对于大多数编程题和面试**：使用简单的切片方法或暴力匹配就足够了
2. **对于文本编辑器**：KMP算法比较合适
3. **对于大文件搜索**：Boyer-Moore算法通常表现最好
4. **对于多模式匹配**：Rabin-Karp算法可以扩展
5. **Python内置方法**：`str.find()`使用了高度优化的C语言实现，通常比任何Python实现的算法都要快

## 总结

字符串匹配是计算机科学中的基础问题，不同的算法有不同的适用场景：
- **暴力匹配**简单直观，适合小规模数据
- **KMP**保证了最坏情况下的线性时间复杂度
- **Boyer-Moore**在实际文本搜索中表现优异
- **Rabin-Karp**适合多模式匹配和流式数据

在实际开发中，应根据具体需求选择合适的算法。对于大多数应用，Python内置的字符串查找方法已经足够高效。
