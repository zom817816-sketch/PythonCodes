# LeetCode 学习代码生成 Skill

## 用途

根据 LeetCode 题目描述，生成结构完整、注释详细、适合学习的 Python 代码文件。

## 核心原则

1. **教学优先**：代码不仅要正确，更要易于理解和学习
2. **循序渐进**：提供从暴力到最优的多种解法，展示优化过程
3. **详细注释**：每个关键步骤都有解释，包含图解示例
4. **深度分析**：末尾提供正确性证明和易错点提示

## 各部分写法规范

### 1. 模块级 Docstring

#### 题目描述
- 包含完整的题目描述，保持原题意不变
- 输入示例和输出示例清晰标注

#### 核心概念
- 如果是系列题目，提供与其他相关题目的对比表
- 如果是新类型题目，解释核心概念和关键洞察
- 使用表格形式展示对比维度，清晰直观

#### 解题思路总览
- 使用表格列出所有解法的复杂度和推荐程度
- 标注推荐解法（⭐⭐⭐⭐）
- 说明每种解法的适用场景

#### 核心思想
- 详细解释主要解法的核心思想
- DP 题目需包含状态定义、状态转移、初始条件、最终答案
- 非 DP 题目需包含算法的关键步骤和原理

### 2. 解法函数

#### 函数命名
- 主函数使用题目要求的函数名（如 LeetCode 原题函数名）
- 辅助解法使用 `<主函数名>_<解法标识>` 的命名方式（如 `maxProfit_dp`, `maxProfit_greedy`）

#### 函数 Docstring
- **核心思想**：简明扼要地说明解法的核心思路
- **算法步骤**：列出清晰的步骤清单
- **复杂度分析**：明确时间和空间复杂度，并解释原因
- **图解示例**：使用具体输入示例，逐步推演算法执行过程
- **结果验证**：标注预期结果，并使用 ✓ 标记正确

#### 代码实现
- 每行关键代码都有行内注释
- 变量命名清晰，符合 Python 规范
- 特殊逻辑有注释说明
- 边界条件有处理
- 代码风格统一

### 3. 正确性证明和易错点

#### 正确性证明
- 陈述定理
- 分点证明（状态完备性、状态转移正确性、全局最优性等）
- 使用数学归纳法或逻辑推理

#### 易错点提示
- 列出常见错误场景
- 说明错误原因和正确做法
- 提供反例或具体例子

## 自动修复规则

1. **类型注解**：将 `List[int]` 改为 `list[int]`（Python 3.9+ 标准）
2. **空数组处理**：在函数开头添加空数组特判
3. **边界条件**：处理长度为 0、1 的特殊情况
4. **代码风格**：统一使用 4 空格缩进，符合 PEP 8 规范
5. **中文注释**：所有注释使用中文，便于理解
6. **文件命名**：使用 `<序号>_<题目名称>.py` 的格式

## 使用方法

1. 获取 LeetCode 题目描述
2. 根据题目类型选择合适的解法
3. 按照文件结构 Schema 生成代码
4. 添加详细注释和图解示例
5. 编写正确性证明和易错点提示
6. 验证代码正确性，但最终的代码文件不要添加测试代码

## 注意事项

1. **不要添加不必要的依赖**：尽量使用 Python 标准库
2. **保持代码可运行**：确保代码可以直接运行测试
3. **注释不要过多**：只注释关键步骤，避免冗余
4. **保持风格一致**：统一的命名、缩进、注释风格
5. **教学价值优先**：代码不仅要正确，更要易于学习和理解
